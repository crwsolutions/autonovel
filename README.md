# AutoNovel

An autonomous pipeline for writing, revising, and exporting complete novels. From a seed concept to finished prose — all generated by AI agents using a five-layer co-evolution system.

**Technology:** .NET 10 (C#), OpenAI-compatible API (llama.cpp), xUnit

---

## Quick Start

```bash
# Setup
cd src/Autonovel
dotnet restore
cp appsettings.json.example appsettings.json  # Configure API endpoint

# Generate foundation docs (world, characters, outline)
dotnet run -- foundation

# Draft chapter N
dotnet run -- draft 5

# Revision cycles (adversarial editing + reader panel)
dotnet run -- revise --max-cycles 5

# Export manuscript
dotnet run -- export

# Opus Review Loop (Phase 3b - manual)
dotnet run -- review
```

## Architecture

### Five Co-Evolving Layers

```
Layer 5: voice.md       — Style guardrails + discovered exemplars
Layer 4: world.md       — Lore, magic system, geography, history
Layer 3: characters.md  — Registry with wound/want/need/lie chains
Layer 2: outline.md     — Beats + foreshadowing ledger
Layer 1: chapters/ch_*.md — The prose
Cross-cutting: canon.md — Hard facts database (consistency source of truth)
```

Changes propagate both directions. State tracked in `state.json`.

### Keep/Discard Workflow

Every generation is evaluated immediately:
- **Foundation:** Keep if `foundation_score > 7.5 AND lore_score > 7.0`
- **Drafting:** Keep if `score > 6.0` (forward progress over perfection)
- **Revision:** Stop when scores plateau (Δ < 0.3 across 2 cycles)

Git workflow: Commit after every successful evaluation. `git reset --hard HEAD~1` on regression.

## Commands

### `foundation`
Generates world bible, character registry, outline, and canon from seed concept.
```bash
dotnet run -- foundation
```

### `draft <chapter-number>`
Drafts a single chapter with full context (voice, world, characters, outline, canon, prev chapter).
```bash
dotnet run -- draft 5
```

### `revise [--max-cycles N]`
Adversarial revision loop:
1. Adversarial edit pass (identify cuts)
2. Reader panel (4 personas: Editor, Genre Reader, Writer, First Reader)
3. Consensus detection
4. Targeted revisions

Stops when plateau detected or max cycles reached.
```bash
dotnet run -- revise --max-cycles 5
```

### `export`
Concatenates all chapters into `manuscript.md`.
```bash
dotnet run -- export
```

### `review`
Opus Review Loop (Phase 3b). Sends full manuscript to dual-persona reviewer (literary critic + professor of fiction). Parses feedback, applies fixes, repeats until:
- ≥4.5 stars with no major items, OR
- ≥4.0 stars with >50% qualified items, OR
- ≤2 items total

```bash
dotnet run -- review
```

### `test-slop <text>`
Test mechanical slop detection on sample text.
```bash
dotnet run -- test-slop "We must delve into the depths..."
```

## How It Works

### Mechanical Slop Detection

Regex-based detection (no LLM):
- **Tier 1:** Corporate buzzwords (utilize, leverage, facilitate, capitalize, etc.)
- **Tier 2:** Weak verbs (was able to, began to, started to, continued to)
- **Tier 3:** Intensifiers (quite, rather, fairly, somewhat, extremely, incredibly)
- **Fiction AI Tells:** "Little did she know", "not just X but Y", "twinkled with wisdom"
- **Structural Tics:** Uniform sentence length, em-dash overuse, repetitive openers
- **Show-Don't-Tell:** "She was angry" vs physical manifestation

Penalty applied to evaluation scores.

### Evaluation

Two-phase evaluation:
1. **Mechanical:** Slop score calculated, penalty applied
2. **LLM Judge:** Scores voice adherence, beat coverage, character consistency, pacing, show-don't-tell, sensory detail, dialogue quality, tension, prose quality

Returns three weakest sentences, three strongest sentences, and new canon entries.

### Revision Engine

1. **Adversarial Edit:** "Cut 500 words" analysis with classified cuts (FAT, REDUNDANT, OVER-EXPLAIN, SLOP, SHOW-DONT-TELL, TELLING, TENSION, PACING, VOICE, CANON)
2. **Reader Panel:** 4 personas evaluate full manuscript
   - Editor (craft-focused)
   - Genre Reader (fantasy expectations)
   - Writer (mechanics-focused)
   - First Reader (layperson)
3. **Consensus Detection:** Finds chapters flagged by multiple readers
4. **Targeted Revisions:** Top 5 issues per cycle

## Anti-Patterns

**Stability Trap:** AI favors stability over change. Actively fight:
- Characters must end truly different
- Let bad things stay bad
- Withhold information; maintain mystery
- Irreversible decisions and loss

**Slop Patterns:**
- OVER-EXPLAIN (~32% of cuts) — Narrator explains what scenes showed
- REDUNDANT (~26%) — Same insight restated 3-4 times
- Tier-1 banned words
- Em-dash overuse
- Sentence-length uniformity
